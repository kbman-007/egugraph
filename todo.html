1. 产品与范围

  • 锁定 MVP：Web 端、0580、学生/家长/后台三入口。
  • 跑通主闭环：注册、20 题诊断、图谱、推荐、家长报告。
  • 诊断先支持选择题、数值题、简单公式题；练习预留全题型。
  • 明确商业模式：免费诊断、订阅练习、付费报告、家庭/机构账号。

2. 设计文档

  • 补齐 PRD、概要设计、详细设计、数据库设计、API 设计。
  • 固化技术栈：React、Redux、TypeScript、FastAPI、Postgres、Neo4j、Object Storage。
  • 明确 LLM 只做离线内容生产，运行时不依赖 Agent。
  • 制定验收标准、权限矩阵、导入校验、激活和回滚流程。

3. UI/UX

  • 学生端：登录、诊断、答题、练习、hint、讲解、知识图谱。
  • 家长端：诊断报告、目标差距、弱项解释、两周计划、付费入口。
  • 后台端：题库导入、导入报告、题目预览、图谱映射、老师审核。
  • 设计数学公式、图形、几何作图、解释题等输入组件。

4. 题库数据处理

  • 解析 2015-2025 的 616 个 PDF，建立 paper index 和 QP/MS 配对。
  • 自动切题：题干、题图、小问、分值、页码、answer type。
  • 抽取 mark scheme：M1/A1/B1、答案、可接受形式、常见错误。
  • 生成标准题库包：questions、question_parts、mark_schemes、hints、mappings、assets。
  • 做 schema、checksum、引用完整性、覆盖率、hint 泄题校验。

5. 四层图谱

  • 固化 Knowledge Graph：72 个知识点、153 条前置边。
  • 建立 Competency Graph、Skill Graph、Exam Graph。
  • 每道题映射到 Question → Competency → Skill → Knowledge。
  • Neo4j 存图谱关系，Postgres 存题目正文、版本、作答、报告等事实。

6. 判分与学习引擎

  • 实现选择、数值、公式、表格等确定性判分。
  • 实现代数等价、方程/不等式、坐标/函数图判分。
  • 几何作图保存结构化对象，不能只存截图。
  • 解释/证明题使用 mark scheme bullets、规则和 MiniLM。
  • 建立掌握度与推荐引擎：score、confidence、复习间隔、前置知识。

7. 后端开发

  • FastAPI 模块：auth、students、questions、diagnostics、attempts、grading、mastery、recommendations、reports、admin import。
  • Postgres 表：用户、学生、题库、导入批次、作答、判分、掌握度、推荐、报告、支付、审计。
  • Neo4j 查询：题目路径、弱项路径、前置知识、推荐题目。
  • Job runner：题库导入、图谱投影、报告生成、golden tests、数据巡检。

8. 前端开发

  • 建立 React 项目和 typed API client。
  • 开发学生诊断与练习主流程。
  • 开发家长报告页和后台导入审核台。
  • 实现题目渲染器，根据 answer_type 动态加载输入组件。
  • 做 E2E：诊断、图谱、推荐、报告、后台激活。

9. 部署上线

  • 购买域名，规划 www、api、admin 子域。
  • 部署前端 CDN、FastAPI 容器、Worker、Postgres、Neo4j、对象存储。
  • 配置 HTTPS、环境变量、secret、备份、日志、监控、告警。
  • 建立 staging/prod 环境，接入微信/支付宝/Stripe 等支付。

10. 运营与合规

  • 处理内容版权、来源追踪、老师审核记录。
  • 落实学生隐私、家长授权、数据导出/删除。
  • 建立错题申诉、客服、退款、机构合作流程。
  • 做数据看板：诊断完成率、练习留存、续费率、低置信判分率。
